<objective>
Build the deterministic anomaly rule engine — the genuinely novel, LLM-free core of this phase — as pure TypeScript functions plus the `anomaly_alerts` repository that supports insertion, same-day dedup, and date-range queries. The engine owns type, severity, and confidence (D-02/D-03); the LLM is never involved in the decision.

Purpose: Provides the safety-critical decision layer that later plans wrap with LLM narration and push delivery. Keeping it pure (no Groq/db imports in the rule service) enforces D-03's hard boundary by construction and makes it fully unit-testable.
Output: `anomalyRule.service.ts` (4 rule functions), `anomalyAlert.repository.ts`, and missed-schedule count helpers on the medication/dialysis repositories. Turns `anomalyRule.service.test.ts` GREEN.
</objective>

    - checkCAPDEffluentAbnormal(kondisiKeluar): null if null/"jernih"; else {tipeAnomali:"kondisi_cairan_abnormal", severity:"tinggi", confidenceScore:100}. No history requirement (D-04).
    - checkMissedSchedules(missedCountToday): null if <=2; else {tipeAnomali:"jadwal_terlewat", severity:"tinggi", confidence 60+15*(n-2) capped 100}.
    - checkFluidIntakeDeviation(todayIntake, sevenDayAvg): null if <7 days baseline (D-04, D-01 personal baseline); else fires normal-severity when deviation exceeds threshold, confidence scales with deviation.
  </behavior>
