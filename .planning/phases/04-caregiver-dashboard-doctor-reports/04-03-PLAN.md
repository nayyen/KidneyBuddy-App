## Phase Goal

**As a** patient preparing for a doctor visit, **I want to** request a backend-aggregated summary of my fluid in/out, medication adherence, and CAPD condition frequency for a date range, **so that** the report shown to my doctor is accurate and computed from my real tracking data.

> Source: ROADMAP Phase 4 Success Criteria 3. Grounded in CONTEXT.md D-04/D-05/D-07/D-09 and 04-RESEARCH Pattern 4.

<objective>
REPORT-01 backend vertical slice: the `GET /api/report` aggregation endpoint that powers the doctor report. Builds the repository (three date-range queries), the service (injectable core + zod validation, turning the 04-01 RED test GREEN), the controller, the route, and the app mount.

Purpose: This is the only genuinely new backend code in Phase 4. Everything the report UI (04-04) renders comes from this endpoint's response shape. The WIB-timezone boundary (Pitfall 4) and the 90-day cap (D-05) are correctness-critical.
Output: report.repository.ts, report.service.ts, report.controller.ts, report.routes.ts, mounted at /api/report.
</objective>

| T-04-06 | Denial of Service | oversized date-range query | mitigate | reportQuerySchema enforces a 90-day max server-side (returns 400) before any DB query runs. |
