## Phase Goal

**As a** caregiver sharing a patient's account on my own device, **I want to** see reminder-schedule changes made on the patient's device reflected on mine within 30 seconds with a clear notification, **so that** I never act on stale therapy timing.

> Source: ROADMAP Phase 4 Success Criteria 1 & 2 (caregiver identical real-time view + cross-device change notification). Grounded in CONTEXT.md D-01/D-02/D-03/D-08.

<objective>
CAREGIVER-02 vertical slice: wire the full cross-device reminder-sync path end to end — DB (`updated_at` mutation) → backend (push fan-out on update) → frontend (30s polling + toast). CAREGIVER-01 is satisfied automatically by the shared-account model (D-08, no role logic); this plan's polling is the only behavior that makes the caregiver's identical view *live*.

Purpose: D-02 change detection only works if `update()` actually bumps `updated_at` (Drizzle does NOT auto-set it — Pitfall 1). D-03's notification only fires if `updateReminder` calls `sendToAllDevices`. Both gaps were verified by research and must be closed together with the polling consumer.
Output: patched repository update, patched controller, polling on the Pengingat page.
</objective>
