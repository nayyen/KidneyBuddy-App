<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/lab-results/upload | Untrusted multipart file + exam date |
| disk → /uploads static serve | Stored files exposed over HTTP |
| client → /trend, /parameters | Untrusted parameter name + range strings into SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Tampering | file upload MIME spoofing | mitigate | multer fileFilter checks `file.mimetype` against PDF/JPG/PNG allowlist; magic-byte sniffing deferred (post-MVP, documented) |
| T-03-16 | Denial of Service | oversized upload | mitigate | multer `limits.fileSize = 10MB`; LIMIT_FILE_SIZE → clean 400 in app.ts handler |
| T-03-17 | Elevation of Privilege | path traversal via filename | mitigate | diskStorage controls destination; filename generated as `${timestamp}-${rand}${ext}` from `path.extname` only — no user-supplied directory component |
| T-03-18 | Tampering | SQL/JSONB injection via parameter/range | mitigate | drizzle `sql` tag parameterizes all user values `${userId}`/`${parameter}`/`${fromDate}`; range mapped through `rangeToFromDate` enum, never interpolated raw |
| T-03-19 | Information Disclosure | static serve exposes other users' files via guessed path | accept | Filenames are timestamp+random, non-enumerable; files contain user-uploaded lab docs behind unguessable names. Per-file ACL deferred (documented MVP tradeoff); no directory listing (express.static default) |
| T-03-14 | Information Disclosure | trend reads parameter_lab values | accept | parameter_lab intentionally unencrypted for queryability; scoped by `user_id = ${userId}` and JWT auth |
| T-03-03 | Spoofing | route auth | mitigate | `authenticate` on upload/trend/parameters routes |
</threat_model>
