<objective>
Build the backend foundation for daily activity logging (ACTIVITY-01, ACTIVITY-03): the `daily_activities` table, repository, service (with Zod validation, WIB timestamp handling, and AES-256-GCM encryption of the feelings note), controller, routes, and route mount. This is the API the activity frontend (03-03) and the pre-end-time push cron (03-02) build on.

Purpose: A user must be able to start an activity with a name + estimated end time, see its current status, and mark it complete with a feelings rating. None of that is possible until this server stack exists.
Output: Working `/api/activities` endpoints (POST, GET, GET /active, PATCH /:id/complete) with the `daily_activities` table pushed to Postgres and green unit tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/activities | Untrusted JSON body (namaKegiatan, estimasiSelesai, perasaan, catatan) crosses here |
| service → Postgres | Encrypted note + activity rows persisted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | GET/PATCH activity by id | mitigate | Repository filters `WHERE user_id = ${userId}` on every query; controllers read `req.user!.id`, never `req.body.userId` (IDOR prevention) |
| T-03-02 | Information Disclosure | catatanPerasaan free-text health note | mitigate | AES-256-GCM encrypt in service before INSERT (existing `encrypt()`); decrypt only on read |
| T-03-03 | Spoofing | All /api/activities routes | mitigate | `authenticate` JWT middleware on every route |
| T-03-04 | Tampering | estimasiSelesai timing (client clock manipulation) | mitigate | Server re-validates combined timestamp is ≥1 min in future using server WIB clock; rejects past times |
| T-03-05 | Denial of Service | reminderSent double-fire (cron) | mitigate | `reminder_sent` boolean column + index added here so 03-02 cron can dedupe; default false |
| T-03-06 | Input Validation | activity payload | mitigate | Zod `createActivitySchema`/`completeActivitySchema` reject malformed/oversized input before persistence |
</threat_model>
