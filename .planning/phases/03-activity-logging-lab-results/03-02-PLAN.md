<objective>
Deliver the pre-end-time push reminder for activities (ACTIVITY-02, decision D-03): a node-cron job that runs every minute, finds berlangsung activities a few minutes before their estimated end time, and sends one push notification through the existing VAPID infrastructure — without ever double-firing.

Purpose: ACTIVITY-02 promises "a reminder a few minutes before estimated end time." Without a persistent, Postgres-queried cron (not in-memory setTimeout), this reminder silently dies on every backend restart. The recurring in-app "Masih aktif" banner (also ACTIVITY-02) is in-app only per D-04 and is built in 03-03 — this plan covers only the single pre-end-time push.
Output: `activityEndReminder.job.ts` mounted in the scheduler, with green dedupe tests.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Denial of Service | repeated pre-end-time push (notification spam) | mitigate | `reminder_sent` flag set after first dispatch; findDue filters reminderSent=false so each activity fires at most once |
| T-03-07 | Tampering | cron fires at wrong local time | mitigate | WIB offset `Date.now()+7*3600*1000` for window math (matches reminderDispatch CR-02 fix); servers default TZ=UTC |
| T-03-08 | Denial of Service | one failing push aborts whole batch | mitigate | Per-activity try/catch logs and continues; batch isolation |
| T-03-03 | Spoofing | push targeting | mitigate | Reuses `sendToAllDevices(userId,...)` which scopes subscriptions to the owning user only |
</threat_model>
