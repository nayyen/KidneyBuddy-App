### Banners (in-app "Masih aktif" notification — D-04)

The "Masih aktif" in-app display is **not a Sonner toast** — it is a persistent inline status rendered inside KegiatanModuleInline (Beranda) and the active ActivityItem (Catatan/Aktivitas). It is updated via `setInterval(60000)` on the client.

It does NOT appear as a push notification (D-04). The pre-end-time push notification (D-03) is a single push via the existing cron/VAPID infrastructure, separate from this in-app display.

### Destructive / Soft-Delete Confirmation

Lab results cannot be permanently deleted (D-08). Archive is the only removal action. Use a neutral confirmation — not red, because this is reversible (user can view archive).

| Action | Dialog Title | Body | Cancel | Confirm |
|--------|-------------|------|--------|---------|
| Archive lab result | "Arsipkan Hasil Lab?" | "Hasil lab ini akan disembunyikan dari daftar utama. Kamu bisa melihatnya kembali di tampilan Arsip." | "Batal" | "Arsipkan" (secondary button, `#2a9d8f` border and text — NOT destructive red) |

**Note:** No permanent delete confirmation exists in Phase 3 — D-08 prohibits hard delete. If a user asks how to permanently delete, show a neutral informational message: "Hasil lab tidak dapat dihapus secara permanen untuk menjaga riwayat kesehatan kamu."
