## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/activities | Client sends activity create/complete payloads (server re-validates — see 03-01) |
| client render | Untrusted activity name displayed in DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Information Disclosure | activity name rendered in list/module | mitigate | Rendered as React text children (auto-escaped); no dangerouslySetInnerHTML anywhere |
| T-03-10 | Tampering | client-side estimasiSelesai "in future" check | accept | Client check is UX only; authoritative re-validation is server-side in 03-01 (T-03-04) |
| T-03-08 | Denial of Service | setInterval leak on unmount | mitigate | Every setInterval has a clearInterval cleanup in its useEffect return |
| T-03-03 | Spoofing | API auth | mitigate | All calls use `authFetch` (bearer access token); unauthenticated calls 401 server-side |
</threat_model>
