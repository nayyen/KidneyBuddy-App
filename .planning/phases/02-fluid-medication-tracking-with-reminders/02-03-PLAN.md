**As a** patient on my phone, **I want to** install KidneyBuddy to my home screen and turn on notifications (with iOS guiding me to Add to Home Screen first), **so that** my device is registered to receive medication and therapy reminders.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→/api/push | Subscription JSON sent from client; must carry the user's auth |
| service worker→/api/medication-log/confirm | SW-initiated POST must include credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | NEXT_PUBLIC_VAPID_PUBLIC_KEY in client | accept | public key is designed to be public; private key stays server-side (02-02) |
| T-02-03-02 | Denial of Service | iOS subscription silently expires, reminders stop | mitigate | visibilitychange re-subscribe + last_confirmed_at refresh per Pitfall 5 |
| T-02-03-03 | Tampering | service worker confirm POST forged | mitigate | SW POST uses credentials include; backend validates reminderId belongs to req.user (enforced in 02-05) |
| T-02-03-SC | Tampering | npm installs (serwist, esbuild) | mitigate | blocking-human legitimacy checkpoint (Task 1) |
</threat_model>
