<objective>
Build the responsive UI shell that every later Phase 2 (and future-phase) screen reuses: three genuinely distinct layouts (mobile bottom-nav / tablet 2-column / desktop sidebar) per D-07/D-08/D-09, and migrate all authenticated pages into an `(app)/` route group so the shell wraps them while `(auth)/` pages stay shell-free.

Purpose: RESPONSIVE-01..04 are foundational — getting the layout system right here means later phases drop screens into the shell without re-deriving navigation or breakpoints.
Output: AppShell + BottomNav + Sidebar + MobileHeader + TopBar + FAB components, an `(app)/` route group with its layout, placeholder pages for catatan/pengingat/edukasi, and an initialized shadcn config.
</objective>

## Phase Goal (user story)

**As a** CKD patient or caregiver using KidneyBuddy on a phone, tablet, or desktop, **I want to** navigate the app through a layout that adapts correctly to my screen size, **so that** the app feels native and usable on whatever device I have at hand.
