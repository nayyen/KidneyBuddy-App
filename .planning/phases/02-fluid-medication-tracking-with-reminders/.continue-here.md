## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| multer single-value string | `fd.append("key", "val")` called once → `req.body.key = "val"` (string), NOT `["val"]` (array). `z.array()` parse fails with ZodError. | advisory | Always use `z.preprocess((v) => Array.isArray(v) ? v : v ? [v] : [], z.array(...))` on backend for any field that may arrive as single or multiple values via FormData/multer. |
| JWT/toast silent failure | `react-hook-form.handleSubmit` silently catches errors thrown in `onSubmit`. User sees nothing. | advisory | Always wrap `onSubmit` body in `try-catch` with `toast.error(msg)` on catch. Never `throw` bare errors in RHF `onSubmit`. |

<decisions_made>

- **multer fix location = backend** (not frontend JSON stringify workaround): `z.preprocess` in `hariAktifSchema` in `reminders.service.ts` — cleaner than changing FormData serialization strategy on frontend.
- **Modal centering = transform approach** (`sm:inset-auto sm:top-1/2 sm:left-1/2 sm:-translate-x-1/2 sm:-translate-y-1/2`): SheetContent `side="bottom"` hardcodes `inset-x-0 bottom-0`; `sm:inset-auto` overrides all four inset values allowing transform centering. `sm:mx-auto` approach only centers horizontally, not vertically.
- **Scrollbar = inner div pattern** (`flex-1 min-h-0 overflow-y-auto` on form div): SheetContent is already `flex flex-col` (shadcn default). Moving overflow to inner div + `flex-1 min-h-0` is the standard CSS trick for overflow in flex child. Gives clean `pr-2` gap from scrollbar.
- **SheetHeader gets `shrink-0`**: Prevents title from compressing when form content is tall inside flex-col parent.
</decisions_made>
