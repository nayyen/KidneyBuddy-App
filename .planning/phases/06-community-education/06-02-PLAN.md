<objective>
Education vertical (backend half): deliver a working, filterable education content
API plus the seed script that fills it with real Bahasa Indonesia articles. After
this plan a user (via API / curl) can browse and filter education content by
therapy method — the EDU-01 capability is real end-to-end at the API layer; the
frontend half (06-03) renders it.

Purpose: EDU-01 requires browsing/filtering education content by active therapy
method. Content is pre-seeded (no admin UI, PRD v0.3 / D-12) and NOT sensitive, so
no encryption applies. Output: repository + service + controller + routes + app.ts
mount + seed-education.ts (real content) + npm seed:education script.
</objective>

## Phase Goal (slice framing)

**As a** CKD patient, **I want to** fetch education articles filtered to my therapy
method from the API, **so that** the app can show me only relevant, trustworthy
guidance.
