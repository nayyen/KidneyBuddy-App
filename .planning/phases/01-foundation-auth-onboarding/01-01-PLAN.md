<objective>
Stand up the mandatory 3-container microservices skeleton (Next.js frontend, Express backend, PostgreSQL database) and prove it works end-to-end with the thinnest possible real vertical slice: a user submits the registration form, the backend hashes the password with Argon2id and writes a real `users` row via Drizzle, and the user lands on a placeholder dashboard that reads their name back from the backend.

Purpose: Establish the REST container boundary, layered backend (Route->Controller->Service->Repository), and design-token foundation BEFORE feature pressure mounts — directly defusing PITFALLS.md Pitfall 6 ("a monolith wearing three Dockerfiles"). This is the Walking Skeleton every later phase builds on.
Output: Working `docker-compose up` stack; registration end-to-end; SKELETON.md architectural decisions realized in code.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Frontend (Next.js) | User-supplied registration form input crosses here (untrusted). |
| Frontend -> Backend (Express REST) | All data crosses the REST boundary; the only path to the DB. |
| Backend -> Database (pg/Drizzle) | Only the backend container holds DATABASE_URL; db port not exposed to frontend network. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | passwordHash storage | mitigate | Store Argon2id hash only; never persist or return plaintext; register response omits passwordHash field (Task 2). |
| T-01-02 | Tampering | registration request body | mitigate | zod-validate payload server-side in auth.service (never trust client zod); parameterized Drizzle queries only (no string SQL). |
| T-01-03 | Spoofing/Elevation | container boundary | mitigate | cors restricted to http://localhost:3000 (no wildcard); db not network-exposed to frontend; no Next.js api route touches DB/Groq (PITFALLS.md Pitfall 6). |
| T-01-04 | Information Disclosure | duplicate-email probe | accept | Register returns 409 on existing email — a minor account-enumeration surface acceptable for MVP; full enumeration hardening deferred (consistent with academic-timeline tradeoff). |
| T-01-SC | Tampering | npm installs (express, drizzle, argon2, pg, zod, next, react-hook-form, shadcn) | mitigate | All packages are from RESEARCH.md/STACK.md HIGH-confidence recommendations verified against npm registry on 2026-06-24; no [ASSUMED]/[SUS]/[SLOP] packages introduced. See note below. |
</threat_model>
