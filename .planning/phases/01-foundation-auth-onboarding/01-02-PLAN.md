<objective>
Complete the authentication vertical slice: persistent login (access + DB-tracked refresh token in an httpOnly cookie), logout, server-side account lockout (AUTH-04) with a live countdown, and independent multi-device login on the same shared account (AUTH-05, the caregiver model).

Purpose: Auth gates every later feature. The lockout must be Postgres-backed (not in-memory) so it survives container restarts (PITFALLS.md security mistakes); tokens must never live in localStorage (XSS). The shared-account multi-device model is the foundation Phase 2's per-device push subscriptions build on.
Output: Working login/logout/lockout; session persists across refresh; two browsers can log into one account independently.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Backend (login) | Untrusted credentials; brute-force surface. |
| Cookie storage (refresh token) | httpOnly cookie not readable by JS — XSS-resistant. |
| Client memory (access token) | Short-lived; lost on tab close (acceptable — refresh restores). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Elevation of Privilege | login brute force | mitigate | Postgres-backed account lockout: 5 fails/10 min -> 15 min lock, server-enforced, survives restart (AUTH-04, not MemoryStore). |
| T-01-06 | Information Disclosure | token theft via XSS | mitigate | Refresh token in httpOnly+Secure+SameSite=Strict cookie; access token in memory only, never localStorage (STACK.md). |
| T-01-07 | Spoofing | forged/replayed tokens | mitigate | JWT verified server-side with secret; refresh tokens DB-tracked and revocable; logout sets revokedAt. |
| T-01-08 | Repudiation | login attempts | mitigate | Every attempt recorded in login_attempts (succeeded flag + timestamp) for audit and lockout. |
| T-01-09 | Information Disclosure | lockout/login error messaging reveals account existence | accept | Calm lockout message implies the email exists — minor enumeration surface accepted for the 13-week academic MVP (D-12 keeps messaging minimal). |
| T-01-SC | Tampering | npm installs (jsonwebtoken) | mitigate | jsonwebtoken@9.0.x is a STACK.md HIGH-confidence recommendation verified against npm registry 2026-06-24; no [ASSUMED]/[SUS]/[SLOP] packages. |
</threat_model>
