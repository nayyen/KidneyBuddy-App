# Phase 4.1 — UX Polish, Data Consistency & Cuci Darah Compliance

## Goal

Fix critical data-consistency bugs (medication check reverts on reload, time drifts with laptop clock, fluid entry time wrong), add the missing Cuci Darah (HD/CAPD session) compliance feature parity with medication, and apply a comprehensive UX polish pass across `/beranda`, `/catatan`, and `/pengingat` — so that data is permanent and consistent across all pages, times are WIB-correct, late items show clear alerts, and reminders respect their day/toggle filters everywhere.

## Root Causes Already Diagnosed

1. **Medication check reverts on reload** — `medicationLog.service.getTodayLogs()` merges scheduled reminders (always `tertunda`) with real logs without dedup by `reminderId`. Confirmed logs get masked.
2. **Medication time drifts** — `scheduledAsLogs` uses `new Date()` for `waktuPengingat` instead of the reminder's scheduled `jamPengingat`.
3. **Fluid entry time wrong (00:xx at 7am WIB)** — `fluid.service.ts` `defaultWaktu` uses `new Date().getHours()` (container UTC). Need WIB offset.
4. **`findNextUpcoming` ignores `hariAktif`** — returns next reminder by time only, no day-name check; only returns one reminder.

## Wave 0 — Backend Data-Consistency Fixes (BLOCKING, highest impact)

### 04.1-01 — Fix medication log dedup + time + WIB helper

**Files:** `backend/src/services/medicationLog.service.ts`, `backend/src/repositories/medicationLog.repository.ts`, `backend/src/utils/wib.ts` (new)

**Tasks:**
1. Create `backend/src/utils/wib.ts` — shared WIB time helpers:
   - `wibNow(): Date` → `new Date(Date.now() + 7*3600*1000)`
   - `wibDateStr(): string` → WIB `YYYY-MM-DD`
   - `wibHHmm(): string` → WIB `HH:mm`
   - `wibDayName(): string` → Indonesian day name from WIB
   - `wibDateFromHHmm(hhmm: string): Date` → today's WIB date with given time
2. Fix `getTodayLogs()`:
   - Use `findTodayObatReminders(userId, todayDayLower)` (which already excludes reminders with a log today via `NOT IN` subquery) instead of `findActiveObatByUser` + JS filter.
   - For `scheduledAsLogs`, set `waktuPengingat: wibDateFromHHmm(r.jamPengingat)` instead of `new Date()`.
   - Keep real logs from `findTodayByUser`; merge with dedup (logs take priority over scheduled by `reminderId`).
   - Sort merged list by `jamPengingat` ascending.
3. Fix `findTodayByUser` to use WIB date bounds (`wibDateStr()` based start/end) instead of server-local `new Date()` bounds.
4. Fix `_confirmCore` insert: set `waktuPengingat` to `wibDateFromHHmm(reminder.jamPengingat)` if available, else `wibNow()`.
5. Update `markConfirmed` to also set `waktuKonfirmasi: wibNow()` (already does `new Date()` — change to WIB).

**Verify:** Confirm an obat → reload `/beranda` and `/catatan` → stays checked. Time shows reminder's scheduled time, not current clock.

### 04.1-03 — Fix `findNextUpcoming` to respect hariAktif + return grouped

**Files:** `backend/src/repositories/reminderSchedule.repository.ts`, `backend/src/services/reminders.service.ts`, `backend/src/controllers/reminders.controller.ts`

**Tasks:**

   - Return `{ obat: ReminderSchedule | null, cuciDarah: ReminderSchedule | null }` — both groups, each the single next upcoming.
   - If no reminder today for a group, wrap to earliest tomorrow (only if tomorrow's day is in hariAktif).
2. Update `getNextUpcoming` controller to return the grouped shape.
3. Update `PengingatBerikutnyaCard` consumer (Wave 1) to render two sections.

**Verify:** A reminder with no matching day or `aktif=false` never appears in "Pengingat Berikutnya".
