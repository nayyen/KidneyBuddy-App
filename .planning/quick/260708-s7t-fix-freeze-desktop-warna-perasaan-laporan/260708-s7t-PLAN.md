<objective>
Two independent frontend fixes on the KidneyBuddy PWA:

1. **Kill the desktop-only freeze** when the user clicks "Buat Ulang Wawasan" on `/catatan`. This is a REPEAT report: quick-260708-qqd already (a) bounded the backend weekly-insight lab query and (b) added a 45s `AbortController` timeout to `apiFetch`, and production verified `POST /api/ai/weekly-insight/regenerate` returns 200 in ~2s. The freeze STILL happens on the user's Windows laptop (Chrome desktop) but NEVER on her Android phone. Conclusion the executor MUST build on: this is **not** network waiting and **not** the backend — it is heavy synchronous work or a runaway loop on the frontend **main thread** that only manifests at desktop (wide) viewport widths. Diagnosis-first with evidence is MANDATORY; the fix must not re-tread the previous timeout/query work.

2. **Color the Perasaan badges** in the `/laporan` → tabel Aktivitas so each perasaan type gets a distinct color, matching exactly how the `/catatan` Aktivitas tab colors them — by extracting the existing mapping into a shared helper instead of duplicating it.

Purpose: reliability (the freeze makes the core "Wawasan Mingguan" feature unusable on desktop) + report legibility.
Output: a stated root cause with evidence + a minimal frontend fix for the freeze; a shared perasaan helper + colored report badges.

Both fixes are **frontend-only**. No backend source changes are expected — so the backend tsc baseline (4 errors) and test suite (269/272) stay unchanged by construction; run them only to confirm no accidental regression.
