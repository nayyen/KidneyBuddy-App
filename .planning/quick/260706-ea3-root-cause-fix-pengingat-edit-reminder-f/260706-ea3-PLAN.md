<objective>
Fix the /pengingat medication reminder EDIT form disappearing the instant a user picks a photo in the native OS file picker.

Root cause (verified in codebase): `ReminderList.tsx`'s `fetchReminders` calls `setIsLoading(true)` on EVERY refetch. Any background refetch — triggered by window focus (which fires when the OS file picker closes), visibilitychange, the 30s cross-device poll, or SYNC_EVENTS — flips `isLoading` to `true`, causing `ReminderList` to return the skeleton branch (lines 79-91) instead of the `<ReminderItem>` list. This UNMOUNTS every `ReminderItem`, destroying each item's local `showEdit` state. Since `EditReminderSheet` is rendered inside `ReminderItem` gated on `showEdit` (ReminderItem.tsx lines 251-259), the open edit sheet is destroyed and never reopens.

Fix: make the skeleton render only on the true initial load. Background refetches must update `reminders` state in place without ever flipping `isLoading` back to `true` (no subtree teardown).

Purpose: The user's edit-with-photo flow is currently unusable; the sheet vanishes before they can save.
Output: A ReminderList that survives background sync ticks without tearing down the item/sheet subtree.
</objective>
