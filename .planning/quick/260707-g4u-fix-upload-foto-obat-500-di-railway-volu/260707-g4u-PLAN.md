<objective>
Fix two production defects.

1. Photo/file upload returns HTTP 500 (EACCES) on Railway. Railway mounts the persistent
   volume at /app/uploads owned by root, shadowing the image's build-time `chown express:nodejs`.
   Because the container runs as non-root `express` (uid 1001), multer cannot write into the
   mounted directory. Fix: a root-privileged entrypoint chowns the mounted volume at startup,
   then drops privileges to `express` before exec'ing the node process. (Local named volumes
   inherit image ownership on first use, which is why this never reproduced locally.)

2. The "Pengingat Berikutnya" card on /beranda is display-only per the PO. Remove the inline
   confirm checkbox; confirmation must happen only from the "Obat Hari Ini" / "Cuci Darah Hari Ini"
   cards.

Purpose: Restore upload reliability in production and align the beranda card with PO intent.
Output: Updated Dockerfile + new entrypoint script + trimmed PengingatBerikutnyaCard.
</objective>

Create backend/docker-entrypoint.sh as a POSIX sh script (shebang `#!/bin/sh`, `set -e`).

runs `chown -R express:nodejs /app/uploads` (a no-op when already owned, so it is safe for

`"$@"` so the Dockerfile CMD remains the single source of the run command.

Edit backend/Dockerfile runner stage:
- Add `su-exec` to the runner image: `RUN apk add --no-cache su-exec` (place it before/with the
  addgroup/adduser RUN, keep the existing mkdir + chown lines as-is for the non-mounted case).

  the container must start as root to be able to chown the root-owned mounted volume.

- Keep CMD identical: `CMD ["node", "--import", "tsx", "src/server.ts"]` — it becomes the `$@`
  passed to the entrypoint.
