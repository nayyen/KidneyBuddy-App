## Decisions Made

- **Per-user-timezone dispatch over inline query rewrite** — `dispatchDueReminders()` runs one `_dispatchCore` pass per distinct user timezone rather than joining/comparing inline in a single query, preserving the existing `(time, day)` `findDue` contract and `reminderDispatch.test.ts`'s `DispatchDeps` shape.
- **Sumber made genuinely required, not just relabeled** — removing a misleading "(opsional)" label without also fixing the underlying (lax) validation would have made the UI *look* required while still silently accepting a blank value; fixed both together on frontend and backend.
- **Two real bugs found during Task 4's live device test beyond the original code audit** — `upsertByEndpoint` never reassigning `userId` on an endpoint conflict (breaking shared-device/multi-account scenarios), and `NotificationPermissionBanner` never re-registering a device when permission was already granted. Both fixed and covered by new regression tests.
- **Placeholder icons, not a fabricated "final" logo** — confirmed there is no branded logo/icon asset anywhere in the repo (not in `frontend/`, not in the `KidneyBuddy_Design` Figma export); generating a real one is a design task outside this bug-fix quick task's scope. Deferred rather than fabricated.
