<objective>
Fix push notification "Pengingat Diperbarui — Jadwal pengingat telah diperbarui dari perangkat lain." appearing on the SAME device that made the edit. With only one device logged in, the user should receive zero cross-device notifications about their own action.

Root cause: `updateReminder` (backend/src/controllers/reminders.controller.ts:76) calls `sendToAllDevices(userId, ...)` which fans out to EVERY active push subscription for the user — including the device that sent the edit request. The backend has no way to identify the originating device because the frontend never tells it which push endpoint made the request.

Fix: let the originating device identify itself via an `X-Push-Endpoint` request header; exclude that endpoint from the fan-out. Push subscriptions are keyed by `endpoint` (unique per browser/device), so this cleanly targets "all devices EXCEPT the sender."

Purpose: Notifications about "an update from another device" must only reach OTHER devices — the whole point of the CAREGIVER-02 cross-device notice.
Output: Backend exclusion param + header wiring; frontend endpoint helper attached at all reminder-update call sites; extended unit test.
</objective>
