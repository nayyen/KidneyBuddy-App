<objective>
Fix login/register failing on mobile phones. Root cause (confirmed via prod DB evidence): frontend (vercel.app) and backend (railway.app) are cross-site, so the httpOnly `refreshToken` cookie is a THIRD-PARTY cookie that iOS Safari (ITP) and several Android browsers block — server-side login succeeds, but the cookie is never stored, so the next page's `/api/auth/refresh` returns 401 and bounces the user back to login. Desktop Chrome (allows 3rd-party cookies) works, matching the report exactly.

Fix: route all JSON API calls same-origin via Next.js rewrites (`/api/*` on vercel.app → proxied to Railway), making the cookie FIRST-party and accepted by every browser. Multipart uploads and `/uploads` image GETs stay DIRECT to the backend (Bearer auth / query-param token, avoids Vercel proxy body-size limits). Plus email normalization hardening (mobile keyboards add trailing spaces; iPhone autofill capitalizes emails) and a visible login network-error message.

Purpose: Restore the core value — patients must be able to log in and stay logged in on their phones.
Output: 9 edits across 9 files, split into 2 atomic commits.
</objective>
