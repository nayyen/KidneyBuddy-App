<objective>
Fix the timezone bug in `findNextUpcoming` where the "sudah dikonfirmasi hari ini"
exclusion window uses the Node process's local midnight (UTC in the Docker
container) instead of the WIB calendar day. Between 00:00–07:00 WIB this causes
reminders confirmed yesterday-evening WIB to be wrongly hidden from the
"Pengingat Berikutnya" card, and after 07:00 WIB causes early-morning WIB
confirmations to be wrongly counted as belonging to the previous UTC day.

Purpose: The Pengingat Berikutnya card must reflect the patient's WIB calendar
day for what counts as "already confirmed today" — reminder reliability (core
value) breaks when a correctly-confirmed dose reappears, or a still-due dose
disappears, purely because of a UTC vs WIB day boundary.

Output: Two exclusion queries repointed to a WIB-day-anchored window via the
existing `wibDayBounds()` helper, plus a pure unit regression test for the
helper's boundary behavior.
</objective>

Findings from planning investigation (executor: confirm, do not re-derive):

- `wibDayBounds(dateStr?)` ALREADY EXISTS in `backend/src/utils/wib.ts` (approx
  lines 86–94). It returns `{ start, end }` as UTC `Date` objects representing
  00:00:00.000 and 23:59:59.999 of the given WIB calendar day (defaulting to
  today-WIB via `wibDateStr()`). DO NOT add a new helper — import and use this.

- Convention choice: `findNextUpcoming` already derives `currentTime` and
  `todayDay` from the WIB-hardcoded helpers `wibHHmm()` / `wibDayNameLower()`.
  The internally-consistent fix therefore uses `wibDayBounds()` (WIB-anchored),
  NOT the per-user `localDayBounds(timezone)` used by medicationLog.service —
  mixing a per-user window with a WIB-hardcoded currentTime/todayDay would be
  inconsistent. Note in SUMMARY: a future coherent improvement would migrate the
  WHOLE of findNextUpcoming (currentTime, todayDay, AND bounds) to the user's
  stored timezone; that is a larger change, out of scope here. The demo user
  Lukman is on Asia/Jakarta, so both approaches yield identical live results.

- `grep -rn "setHours" backend/src` returns ONLY the two target lines
